# Demo Script

## Goal
Show end-to-end understanding in ~8 minutes.

## Flow
1. Explain the problem this repo solves and the primary constraints.
2. Walk through architecture at subsystem level.
3. Run the main workflow and show expected output.
4. Show one non-trivial bug/risk and how it was addressed.
5. Show tests and what confidence they provide.
6. Close with known limitations and next engineering steps.

## Interview Talking Points
- Why this architecture was chosen over alternatives.
- How failure modes are handled.
- What would change first under higher load/security requirements.
